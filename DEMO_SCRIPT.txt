[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's look at the system architecture. When a user sends a query, it first enters our input_processor and immediately passes through a security_checkpoint node. If the input is safe, it goes to the main orchestrator agent. The orchestrator delegates tasks to two specialized sub-agents: the classification_agent, which identifies materials, and the guidance_agent, which fetches disposal rules. Both sub-agents use Model Context Protocol, or MCP tools, to fetch live rules. If the orchestrator detects hazardous materials, it triggers a human_review pause before generating the final report.

[3:00 — SECURITY & MCP]
Our security checkpoint does more than block injection; it also redacts PII like phone numbers and logs all events to a structured JSON audit stream. The MCP server runs local, lightweight Python functions that act as direct databases for municipal rules.
